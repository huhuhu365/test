服务器端脚本

  • 业务规则（Business Rules） - 用于在记录的插入、更新、删除或查询时执行逻辑。可设为 Before（记录保存前）、After（记录保存后）、Async（异步）或 Display（显示时）。
  • 脚本包括（Script Includes） - 定义可复用的服务器端函数。可以是 客户端可调用的（Client callable）或 服务器端专用的。
  • 计划任务（Scheduled Jobs） - 定时执行脚本，用于定期任务，如数据清理、报告生成等。
  • 工作流脚本（Workflow Scripts） - 在工作流中的脚本活动或脚本条件中使用，控制工作流的行为。
  • 事件脚本（Event Scripts） - 当特定事件触发时执行的脚本，通常与 事件注册表（Event Registry）一起使用。
  • 邮件脚本（Email Scripts） - 定制邮件通知的内容，通常用于邮件模板中。
  • 入口动作（Inbound Actions） - 处理从外部系统发送到 ServiceNow 的电子邮件，执行特定操作。
  • 导入映射脚本（Transform Maps Script） - 在导入数据时转换和映射数据的脚本。

    • 客户端脚本

  • 客户端脚本（Client Scripts） - 用于表单加载（onLoad）、字段值变化（onChange）、表单提交（onSubmit）或字段初始化（onCellEdit）时执行的脚本。
  • UI 答案（UI Actions） - 在表单、列表或相关列表上添加按钮或链接，可执行客户端和服务器端操作。
  • UI 策略（UI Policies） - 动态控制表单字段的可见性、可编辑性和必填性，可以包含客户端脚本。
  • UI 页面（UI Pages） - 创建自定义用户界面，通常用于构建复杂的自定义应用程序或服务门户。
  • UI 宏（UI Macros） - 创建可重用的 HTML 片段，通常用于表单设计或自定义 UI 组件。
  • 服务目录客户端脚本（Catalog Client Scripts） - 用于服务目录项表单中的客户端脚本。
  • 服务目录 UI 策略（Catalog UI Policies） - 动态控制服务目录项表单字段的可见性、可编辑性和必填性。

在 ServiceNow 中，涉及到自定义（カスタマイズ）和定制（カスタマイズ）相关的概念和术语有很多。以下是一些常见的 ServiceNow 自定义和定制相关的内容：

Customization（カスタマイズ）：

  • 客户端脚本（Client Scripts） - 在客户端执行的脚本，用于控制表单字段的行为和验证。
  • 业务规则（Business Rules） - 在服务端执行的脚本，用于在记录插入、更新、删除时触发操作。
  • UI 答案（UI Actions） - 如按钮、链接等，用于触发特定的客户端或服务端脚本。
  • UI 答案脚本（UI Scripts） - 在 ServiceNow 中创建和使用的可复用 JavaScript 代码段。
  • 脚本包含（Script Include） - 在服务端定义的可复用脚本代码，允许在多个地方调用。
  • 数据策略（Data Policies） - 强制数据一致性的规则，用于确保数据输入的准确性。
  • UI 答案政策（UI Policies） - 客户端规则，用于控制表单字段的显示、必填、可编辑等属性。
  • 流式逻辑（Flow Designer） - 用于创建可视化的工作流，以自动化业务流程。

Configuration（設定）：

  • 表（Tables） - 数据存储的基本结构，允许用户定义和管理数据。
  • 字段（Fields） - 表中的列，定义数据的类型和属性。
  • 表单布局（Form Layouts） - 定义表单字段的显示和组织方式。
  • 列表布局（List Layouts） - 定义表中记录在列表视图中的显示方式。
  • 词汇（Dictionaries） - 定义字段的属性和行为，如数据类型、长度、标签等。
  • ACL（访问控制规则） - 用于控制用户对数据和操作的访问权限。

Integration（統合）：

  • REST API 和 SOAP API - 用于与外部系统集成，通过 Web 服务实现数据交换。
  • 中继器（MID Server） - 用于在 ServiceNow 和防火墙内的内部系统之间通信。
  • 导入集（Import Sets） - 用于从外部数据源导入数据。

Personalization（個人化）：

  • 用户偏好（User Preferences） - 允许用户自定义其界面和操作的偏好设置。
  • UI 主题（UI Themes） - 更改平台的外观和感觉。

Service Catalog and Portal Customization（サービスカタログとポータルのカスタマイズ）：

  • 服务目录（Service Catalog） - 自定义服务项、类别、表单和工作流程。
  • 服务门户（Service Portal） - 创建和定制用户界面、页面、部件等。

Performance Analytics and Reporting（パフォーマンスアナリティクスとレポート）：

  • 报表（Reports） - 创建和定制报表，分析和展示数据。
  • 仪表板（Dashboards） - 可视化数据，展示关键绩效指标。
